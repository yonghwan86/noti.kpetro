# AI 업무 알림 서비스 — 사용자 매뉴얼

## 시스템 소개

AI 업무 알림 서비스는 장비 및 자산의 정기 점검을 체계적으로 관리하고, 점검 일정에 따라 담당자에게 자동으로 이메일 알림을 발송하는 시스템입니다.

### 주요 기능
- 장비 등록 및 점검 주기 관리
- 자동 점검 상태 추적 (정상 / 점검예정 / 점검지연)
- 매일 오전 9시 자동 이메일 알림 (Gmail API)
- 역할 기반 접근 제어 (마스터 / 구분 관리자 / 담당자)
- 엑셀 일괄 데이터 관리 (내보내기 / 가져오기)
- 대시보드 현황 시각화

---

## 역할 구분

시스템은 3개의 역할로 운영됩니다. 역할에 따라 접근 가능한 메뉴와 기능이 다릅니다.

| 역할 | 설명 | 접근 가능 메뉴 |
|------|------|----------------|
| **마스터 (Admin)** | 시스템 전체 관리자. 모든 데이터를 조회하고 관리할 수 있습니다. | 대시보드, 장비 관리, 팀 관리(전체), 설정, 로그 |
| **구분 관리자 (Manager)** | 특정 구분(카테고리)을 담당하는 관리자. 담당 구분의 장비와 인원을 관리합니다. | 대시보드, 장비 관리, 팀 관리(일부), 로그 |
| **담당자 (Staff)** | 실제 점검을 수행하는 실무자. 배정된 장비의 점검을 기록합니다. | 대시보드, 장비 관리, 로그 |

---

## 역할별 매뉴얼

각 역할에 해당하는 매뉴얼을 참고하세요.

### [마스터(Admin) 매뉴얼](./manual-admin.md)
시스템 전체를 관리하는 마스터를 위한 매뉴얼입니다. 구분/사용자/장비 관리, 설정, 이메일 알림, 엑셀 활용법을 포함합니다.

### [구분 관리자(Manager) 매뉴얼](./manual-manager.md)
담당 구분의 장비와 인원을 관리하는 구분 관리자를 위한 매뉴얼입니다. 장비 점검, 담당자 배정, 엑셀 업로드를 포함합니다.

### [담당자(Staff) 매뉴얼](./manual-staff.md)
배정된 장비의 점검을 수행하는 담당자를 위한 매뉴얼입니다. 로그인, 점검 수행, 이메일 알림 안내를 포함합니다.

---

## 공통 안내

### 최초 로그인
- 마스터가 시스템에 사용자를 등록하면, 해당 이메일로 로그인할 수 있습니다.
- 처음 로그인 시 비밀번호를 설정하고, 개인정보 수집 및 이용에 동의해야 합니다.
- 필수 동의(성명, 이메일, 부서, 직책)를 하지 않으면 서비스를 이용할 수 없습니다.

### 비밀번호 분실
- 마스터에게 비밀번호 초기화를 요청하세요.
- 초기화 후 다음 로그인 시 새 비밀번호를 설정할 수 있습니다.

### 이메일 알림
- 점검 예정일 7일 전부터 예정 알림, 기한 초과 시 지연 알림이 발송됩니다.
- 알림은 매일 오전 9시(KST)에 자동 발송됩니다.
- 알림을 받지 못하는 경우 스팸 메일함을 확인하거나 마스터에게 문의하세요.
