# #6 - 내 일정 기간 등록 — 백엔드 (최종)

## What & Why
personal_tasks 테이블이 단일 시점(scheduledAt datetime)만 지원해 며칠에 걸친 업무를 표현할 수 없다. scheduledEndAt(date-only, YYYY-MM-DD) 컬럼을 추가하고, morningNotified boolean을 lastMorningNotifiedDate(text, YYYY-MM-DD)로 교체하여 기간 일정 중 매일 모닝 알림이 발송되도록 한다.

## 포맷 전제 (모든 구현에서 일관 적용)
- **scheduledAt** — ISO datetime 문자열 (예: `2025-04-07T09:00:00`). 날짜 비교 시 `toKSTDateStr(scheduledAt)` 변환 필요.
- **scheduledEndAt** — YYYY-MM-DD 날짜 문자열 (예: `2025-04-11`) 또는 `null`. 이미 날짜 형식이므로 변환 없이 문자열 비교 가능.

## Done looks like
- `server/utils.ts` 신규 생성 — `toKSTDateStr`, `formatDateShort` 두 함수 정의, routes.ts와 scheduler.ts 양쪽에서 import
- `shared/schema.ts`에 scheduledEndAt(text nullable), lastMorningNotifiedDate(text nullable) 추가, morningNotified boolean 제거 후 `npm run db:push` 동기화
- **POST** `/api/personal-tasks`: scheduledEndAt 수신, null은 허용하고 문자열일 때만 YYYY-MM-DD 패턴 검증, 종료일 ≥ 시작일(날짜 기준) 검증, 기간 일정이면 repeatType 강제 none, `taskData.scheduledEndAt = req.body.scheduledEndAt ?? null`, 공유 푸시에 `"4/7 ~ 4/11"` 또는 `"4/7"` 형태 메시지

# #7 - 내 일정 기간 등록 — 프론트엔드 (최종)

## What & Why
백엔드에 scheduledEndAt(YYYY-MM-DD) 필드가 추가됨에 따라, MySchedule.tsx의 폼·필터·상태 표시 모두를 기간 일정에 맞게 업데이트한다. 기존 단일 일정 동작은 변경 없음.

## 포맷 전제
- **scheduledAt** — ISO datetime 문자열 (예: `2025-04-07T09:00:00`)
- **scheduledEndAt** — YYYY-MM-DD 날짜 문자열 (예: `2025-04-11`) 또는 `null`
- 오늘 날짜 비교: `format(new Date(), 'yyyy-MM-dd')` — 브라우저 로컬 시간대 기준. 기존 `isToday(parseISO(...))` 동작과 동일 기준으로 통일.
- scheduledAt에서 날짜 추출: `format(parseISO(t.scheduledAt), 'yyyy-MM-dd')` — UTC `split('T')[0]` 금지, parseISO 후 format으로 로컬 시간대 변환.
- `format`은 기존 import(14행)에 이미 포함 — 추가 import 불필요.

### 5. getScheduleStatus() 수정 (177-184행)

```typescript
const getScheduleStatus = (task: TaskWithShared) => {
```

### 6. 기간 일정 카드 날짜 표시 (247행)

현재:

변경:

> 단일 일정의 포맷(`'yyyy-MM-dd HH:mm'`)은 기존 그대로 유지.

formatDateShort 로컬 함수 (컴포넌트 상단에 정의):
```typescript
const formatDateShort = (dateStr: string) => {
  const [, m, d] = dateStr.split('-');
  return `${parseInt(m)}/${parseInt(d)}`;
};
```
