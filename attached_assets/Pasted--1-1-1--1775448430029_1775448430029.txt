개선 방향은 두 가지입니다.

첫째, 1인 1일 최대 1통 원칙으로 다이제스트를 통합합니다. 장비 점검 알림과 개인일정 알림을 사용자별로 모아 하나의 이메일로 발송합니다.

둘째, 푸시를 주 채널로, 이메일을 보조 채널로 전환합니다. 즉시성이 필요한 알림(10분 전 리마인더, 공유 일정 생성)은 푸시로만 발송하고, 이메일은 하루 1회 다이제스트로만 사용합니다.

추가로, 모닝 다이제스트 발송 시간을 9AM에서 6AM으로 변경합니다. 일찍 출근하는 직원들이 출근 전에 알림을 확인할 수 있도록 합니다.

Done Looks Like
6AM에 사용자별 통합 다이제스트 이메일 1통 발송 (장비 점검 알림 + 오늘 일정 + 내일 예정 일정 포함)
기존 6PM 다이제스트는 제거 — 9AM(→6AM) 다이제스트의 "내일 예정" 섹션으로 통합
10분 전 리마인더는 푸시만 발송 (기존과 동일)
공유 일정 생성 시 푸시만 발송 (기존과 동일)
일일 이메일 발송량이 최대 400통 이하 (일정이 없는 사용자에게는 미발송)
기존 알림 누락 없음 — 모든 알림 내용이 다이제스트 또는 푸시를 통해 전달됨
서버 재시작 시 catch-up 로직 정상 동작
Out of Scope
이메일 인프라 교체 (Gmail → SES 등은 추후 별도 이슈)
사용자별 알림 설정 (A3 — 추후 별도 이슈)
발송 큐 도입 (추후 별도 이슈)
개인일정 기간 등록 (Phase 1 — 본 이슈 완료 후 진행)
이메일 HTML 템플릿의 전면 리디자인
Tasks
Task 1. 스케줄러 cron 시간 변경

0 9 * * * → 0 6 * * * (6AM KST)로 변경합니다. 이 cron에서 실행되는 함수들의 호출 구조를 다이제스트 패턴으로 재구성합니다. 서버 재시작 catch-up 로직의 시간 기준도 kstHour >= 9 → kstHour >= 6으로 변경합니다.

Task 2. 사용자별 알림 수집 함수 구현

새로운 함수 collectDailyDigestForUser(userId)를 구현합니다. 이 함수는 한 사용자에 대해 다음 내용을 수집합니다:

장비 점검 알림: 담당(staff) 장비, 관리(manager) 장비, 팀장으로서 받는 장비 중 점검 예정(7일 이내) 및 기한 초과 건
오늘의 개인일정: 본인 일정 + 공유받은 일정 중 오늘 예정인 것
내일 예정 일정: 본인 일정 + 공유받은 일정 중 내일 예정인 것 (기존 6PM 다이제스트 대체)
수집 결과가 모두 비어 있으면 해당 사용자에게는 이메일을 보내지 않습니다.

Task 3. 통합 다이제스트 이메일 템플릿 구현

하나의 이메일 안에 섹션별로 내용을 표시합니다:

Copy[오늘의 알림 요약] — 2026-04-07(월)

📋 점검 예정 장비 (3건)
  • A장비 — 4/10 예정 (담당: 김OO)
  • B장비 — 4/8 예정 (담당: 이OO)
  • C장비 — 기한 초과 (3/28 만료, 담당: 박OO)

📅 오늘 일정 (2건)
  • 10:00 팀 회의
  • 14:00 출장 보고서 제출

📅 내일 예정 (1건)
  • 09:00 상반기 장비 점검 시작
수집된 섹션만 표시합니다 (점검 알림이 없으면 해당 섹션 생략).

Task 4. 기존 개별 발송 함수 정리

기존 checkUpcomingInspections()의 발송 로직을 분리합니다. 수신자 수집 로직(collectRecipients)은 그대로 유지하되, 이메일 발송 부분을 다이제스트 수집기로 연결합니다. 푸시 발송(collectPushRecipientIds → sendPushToUser)은 기존과 동일하게 유지합니다.

기존 checkPersonalTasksMorning()의 이메일 발송 부분도 다이제스트 수집기로 연결합니다. 푸시 발송은 기존과 동일하게 유지합니다.

Task 5. 6PM 다이제스트 cron 제거/통합

sendOwnerTomorrowDigest()의 "내일 예정" 내용을 Task 2의 수집 함수에 통합합니다. sendSharedTasksEmailDigest()의 공유 일정 내용도 통합합니다. 기존 0 18 * * * cron에서 이 두 함수 호출을 제거합니다. 해당 cron에 다른 작업이 남아 있지 않다면 cron 자체를 제거합니다. catch-up 로직에서 kstHour >= 18 블록도 정리합니다.

Task 6. 멱등성(idempotency) 키 정리

기존 last_email_date와 last_owner_digest_date를 하나의 last_daily_digest_date로 통합합니다. 6AM 다이제스트 발송 완료 후 이 값을 기록하여 중복 발송을 방지합니다. 서버 재시작 catch-up 시 이 키를 확인하여 당일 미발송이면 실행합니다.

Task 7. 동작 검증

6AM cron 트리거 시 사용자별 1통의 다이제스트 이메일이 발송되는지 확인
점검 알림 + 오늘 일정 + 내일 예정이 하나의 이메일에 모두 포함되는지 확인
알림 대상이 없는 사용자에게는 이메일이 발송되지 않는지 확인
10분 전 리마인더가 푸시로만 발송되는지 확인
공유 일정 생성 시 푸시로만 발송되는지 확인
서버 재시작 시 6AM 이후라면 catch-up으로 당일 미발송 다이제스트가 발송되는지 확인
기존 6PM 시점에 별도 이메일이 발송되지 않는지 확인
서버 로그에 발송 에러가 없는지 확인
