Task 3. PATCH 핸들러 ✅ 정확, 1개 엣지케이스

scheduledEndAt이 포함된 업데이트라면 updateData.repeatType = 'none' 강제 (null→단일 복귀 시는 제외)

이 조건이 좋습니다. 다만 역방향 케이스도 확인: 이미 기간 일정(scheduledEndAt 있음)인 task에서 repeatType: 'weekly'만 보내는 PATCH 요청이 오면? 현재 스펙은 scheduledEndAt이 업데이트에 포함될 때만 강제하므로, 기존 기간 일정의 repeatType을 바꾸는 요청이 통과될 수 있습니다.
해결: PATCH에서 최종 상태의 scheduledEndAt을 확인하는 조건이 필요합니다:
typescriptconst finalEndAt = updates.scheduledEndAt !== undefined 
  ? updates.scheduledEndAt 
  : task.scheduledEndAt;
if (finalEndAt) updates.repeatType = 'none';
이 로직을 스펙에 반영하세요.

Task 6. 기간 일정 카드 날짜 표시 ⚠️ 위치 불명확

renderTask() (186행~) 안에서 날짜 표시 부분에

현재 renderTask의 날짜 표시 코드가 잘린 상태(212행 이후 truncated)라 정확한 삽입 위치를 확인하려면 해당 부분을 봐야 합니다. 잘린 부분을 확인해보겠습니다:코드 위치를 파악하고 범위 표시 수정을 검토했다코드 위치를 파악하고 범위 표시 수정을 검토했다확인했습니다. 날짜 표시는 247행입니다:
typescript{format(parseISO(task.scheduledAt), 'yyyy-MM-dd HH:mm', { locale: ko })}
스펙의 Task 6에서 이 줄을 교체해야 합니다. 스펙에 **정확한 행번호(247행)**를 명시하면 좋겠습니다. 현재 "186행~"이라고만 되어 있어서 찾기 어렵습니다.
교체 로직 자체는 맞습니다:
typescript{task.scheduledEndAt

다만 기존 포맷이 'yyyy-MM-dd HH:mm'인데 단일 일정 변경 후 포맷이 'M/d HH:mm'으로 바뀝니다. 이게 의도적인 변경인지 확인하세요. 의도가 아니면 format(parseISO(task.scheduledAt), 'yyyy-MM-dd HH:mm', { locale: ko })를 유지해야 합니다.
