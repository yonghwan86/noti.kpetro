#8 - 일정 컬러 라벨 + 우선순위 — 백엔드 (최종 확정본)
What & Why

일정 목록에서 일정의 성격과 긴급도를 한눈에 파악하기 어렵다.
label(카테고리 라벨)과 priority(우선순위) 컬럼을 추가하여 시각적 구분과 정렬 기준을 제공한다.

라벨 정의
값	표시명	용도
inspection	장비 점검	점검 관련 업무
meeting	회의	회의·미팅
trip	출장	외근·출장
training	교육	교육·연수
other	기타	분류 외 업무
null	(없음)	라벨 미지정 — 기존 일정 하위 호환
우선순위 정의
값	표시명
0	보통 (기본값)
1	낮음
2	높음
3	긴급
Done looks like
shared/schema.ts에 label(text, nullable), priority(integer, default 0) 추가
insertPersonalTaskSchema에서 label, priority 입력 허용
POST: label, priority 수신. label은 허용값만 유지하고 그 외 값은 null로 정규화한다. priority는 Number() 변환 후 0~3 범위의 정수만 유지하고, 그 외 값은 0으로 정규화한다.
PATCH: label, priority 수신 시 POST와 동일한 기준으로 정규화한다. undefined면 무시하고, label은 허용값 외 null, priority는 범위 외 0으로 처리한다.
GET: 스키마 변경으로 응답 데이터에 자동 포함된다.
Out of scope
라벨별 필터링 API
커스텀 라벨 생성
구현 전 필수 확인

수동 interface 구조라면 프론트에서 사용하는 PersonalTask 타입에 label, priority를 반드시 추가한다. 현재 MySchedule.tsx는 PersonalTask를 @/lib/types에서 import하는 구조다.

const renderDescription = (task: TaskWithShared) => {
  const desc = task.description || '';
  const hasChecklist = /^- \[[ x]\] /m.test(desc);

  if (!hasChecklist) {
    return <p className="text-xs text-muted-foreground truncate">{desc}</p>;
  }

  const lines = desc.split('\n');
  const isOwner = task.userId === currentUser?.id;

  const toggleChecklistItem = (lineIndex: number) => {
    if (!isOwner) return;

    const newLines = lines.map((line, i) => {
      if (i !== lineIndex) return line;
      if (line.startsWith('- [ ] ')) return line.replace('- [ ] ', '- [x] ');
      if (line.startsWith('- [x] ')) return line.replace('- [x] ', '- [ ] ');
      return line;
    });

    checklistMutation.mutate({
      id: task.id,
      data: { description: newLines.join('\n') },
    });
  };

  return (
    <div className="space-y-0.5">
      {lines.map((line, i) => {
        const unchecked = line.match(/^- \[ \] (.+)/);
        const checked = line.match(/^- \[x\] (.+)/);

        if (unchecked) {
          return (
            <div key={i} className="flex items-center gap-1.5 text-xs text-muted-foreground">
              <input
                type="checkbox"
                checked={false}
                onChange={() => toggleChecklistItem(i)}
                disabled={!isOwner}
                className="h-3 w-3 rounded border-gray-300"
              />
              <span>{unchecked[1]}</span>
            </div>
          );
        }

        if (checked) {
          return (
            <div key={i} className="flex items-center gap-1.5 text-xs text-muted-foreground">
              <input
                type="checkbox"
                checked={true}
                onChange={() => toggleChecklistItem(i)}
                disabled={!isOwner}
                className="h-3 w-3 rounded border-gray-300"
              />
              <span className="line-through">{checked[1]}</span>
            </div>
          );
        }

        if (line.trim()) {
          return <p key={i} className="text-xs text-muted-foreground">{line}</p>;
        }

        return null;
      })}
    </div>
  );
};

설명:

체크리스트가 아닌 일반 설명은 기존 truncate 표시를 유지한다.
체크박스 클릭 시 description 전체를 PATCH로 저장한다.
공유받은 일정(isOwner=false)은 체크박스를 disabled 처리한다.
